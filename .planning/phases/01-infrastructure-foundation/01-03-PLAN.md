| Boundary | Description |
|----------|-------------|
| Developer-controlled mailbox | Verification email arrives outside AWS; control of this inbox proves sender ownership |
| AWS SES identity store | Identity status changes from pending to verified after mailbox action |
| AWS Support case | Manual support request; must avoid including subscriber PII |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-05 | Spoofing | Temporary sender email | mitigate | Use only an email address the developer controls; verification link proves mailbox ownership |
| T-01-06 | Tampering | SES suppression settings | mitigate | Keep suppression enabled while switching from domain identity to email identity |
| T-01-07 | Repudiation | SES production access request | mitigate | Record ticket number in `ses-ticket.txt`; include in plan SUMMARY for audit trail |
| T-01-08 | Elevation of Privilege | SES sandbox exit | accept | Production access is legitimate for confirmed opt-in transactional mail; risk is reduced by double opt-in + suppression |
| T-01-SC | Tampering | npm installs | accept | No new package installs in this plan |
</threat_model>
