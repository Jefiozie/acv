  <action>
    Replace the skeleton body of `mock-api.interceptor.ts` with complete mock response logic. The interceptor receives `req: HttpRequest&lt;unknown&gt;` and `next: HttpHandlerFn`.

    Guard at the top: if `!environment.useMockApi` return `next(req)` immediately. All mock logic below this guard runs only in development.

    Use `req.method` and `req.url` to route to the correct mock response. Use `of(new HttpResponse({ status: ..., body: ... }))` from `rxjs` and `HttpResponse` from `@angular/common/http` to return mock responses. Import `delay` from `rxjs/operators` and pipe a `delay(400)` onto each mock response to simulate network latency — this makes the loading states visible during manual testing.

    Mock route: `POST /api/subscribe`
    - Condition: `req.method === 'POST' && req.url.includes('/api/subscribe')`
    - Maintain a `Set&lt;string&gt;` of already-subscribed emails as a module-level variable (persists across requests within the same page session). Read the email from `req.body` (cast as `{ email: string }`).
    - If the email is already in the set: return `new HttpResponse({ status: 409, body: { message: 'Already subscribed' } })`.
    - Otherwise: add the email to the set and return `new HttpResponse({ status: 200, body: null })`.

    Mock route: `GET /api/confirm`
    - Condition: `req.method === 'GET' && req.url.includes('/api/confirm')`
    - Read token from `req.params.get('token')`.
    - If token equals `'invalid'` or `'expired'`: return `new HttpResponse({ status: 400, body: { message: 'Invalid token' } })`.
    - Otherwise (any non-empty token): return `new HttpResponse({ status: 200, body: null })`.

    Mock route: `GET /api/unsubscribe`
    - Condition: `req.method === 'GET' && req.url.includes('/api/unsubscribe')`
    - Always return `new HttpResponse({ status: 200, body: null })` (unsubscribe always succeeds in mock).

    Fallback: if none of the above conditions match, return `next(req)` (pass through to real network; useful if Phase 2 backend is running locally via proxy).

    Import `of` from `rxjs`; `delay` from `rxjs/operators`; `HttpInterceptorFn`, `HttpRequest`, `HttpHandlerFn`, `HttpResponse` from `@angular/common/http`; `environment` from `../../environments/environment`.

    Verify all HTTP calls in `SubscribeComponent`, `ConfirmComponent`, and `UnsubscribeComponent` use relative paths beginning with `/api/` — run a grep to confirm no `http://`, `https://`, or `environment.apiUrl` concatenation appears in these component or service files. Relative paths are required so the mock interceptor URL matching works and so CloudFront proxying works in production.

    Run `cd frontend && npm run test` to confirm all existing specs still pass after interceptor update.
