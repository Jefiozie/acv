    Angular 17+ does NOT generate `src/environments/` automatically. Run `cd frontend && npx ng generate environments` — or create the files manually if the command is unavailable:

    Create `frontend/src/environments/environment.ts` exporting a constant named `environment` with fields `production: true`, `apiUrl: ''` (empty string — all API calls use relative paths so CloudFront proxies them correctly), and `useMockApi: false`.

    Create `frontend/src/environments/environment.development.ts` exporting the same `environment` constant with `production: false`, `apiUrl: ''`, and `useMockApi: true`.

    Ensure `angular.json` has a `fileReplacements` entry in the `development` configuration that swaps `environment.ts` with `environment.development.ts` (ng CLI does this automatically when `ng generate environments` is used; add manually if missing).

    Create `frontend/src/app/interceptors/mock-api.interceptor.ts`: export a functional `HttpInterceptorFn` named `mockApiInterceptor`. At the top of the function body, check `environment.useMockApi` — if false, immediately `return next(req)` (pass through). The full mock responses are wired in plan 03-04; for now the function body just checks the flag and falls through to `next(req)` regardless, so the scaffold compiles and tests pass. Import `HttpInterceptorFn`, `HttpRequest`, `HttpHandlerFn` from `@angular/common/http`, and `environment` from `../../environments/environment`.

    Create `frontend/src/app/app.config.ts`: import `provideRouter` and `withComponentInputBinding` from `@angular/router`; `provideHttpClient` and `withInterceptors` from `@angular/common/http`; `routes` from `./app.routes`; `environment` from `../environments/environment`; `mockApiInterceptor` from `./interceptors/mock-api.interceptor`. Export `appConfig` (type `ApplicationConfig`) with `providers` array containing: `provideRouter(routes, withComponentInputBinding())` and `provideHttpClient(...(environment.useMockApi ? [withInterceptors([mockApiInterceptor])] : []))`. Note: `withComponentInputBinding()` is required for query params to be bound as component `@Input()` — this is how `?token=X` reaches ConfirmComponent and UnsubscribeComponent.

    Create `frontend/src/app/app.routes.ts`: export `routes` typed as `Routes`. Define four routes:
    - `{ path: '', loadComponent: () => import('./subscribe/subscribe.component').then(m => m.SubscribeComponent) }`
    - `{ path: 'bevestigen', loadComponent: () => import('./confirm/confirm.component').then(m => m.ConfirmComponent) }`
    - `{ path: 'uitschrijven', loadComponent: () => import('./unsubscribe/unsubscribe.component').then(m => m.UnsubscribeComponent) }`
    - `{ path: 'privacy', loadComponent: () => import('./privacy/privacy.component').then(m => m.PrivacyComponent) }`
    - `{ path: '**', redirectTo: '' }`

    These import paths will produce TypeScript errors at compile time until plans 03-02 and 03-03 create the component files. This is expected. To allow the scaffold to build before those plans run, add a comment on the route entries: `// TODO: component created in plan 03-02/03-03` and temporarily set `loadComponent` entries to lazy-load a placeholder. Alternatively, verify that `tsc --noEmit` is not run until after 03-02 and 03-03 complete — the build check is in plan 03-04.

    Replace `frontend/src/app/app.component.ts` with a standalone `AppComponent`: decorated with `@Component`, `standalone: true`, `changeDetection: ChangeDetectionStrategy.OnPush`, imports `[RouterOutlet, RouterLink]`. Template contains: a `<header>` with a `<nav>` containing an anchor `<a routerLink="/">ACV Aanhanger Beschikbaarheid</a>`; a `<main>` containing `<router-outlet />`; a `<footer>` containing `<a routerLink="/privacy">Privacybeleid</a>`.

    Update `frontend/src/main.ts` to call `bootstrapApplication(AppComponent, appConfig)` — the CLI may have already generated this; verify it imports from `./app/app.config` not from a now-deleted AppModule.

    Update `frontend/src/app/app.component.spec.ts` (the Vitest smoke test): import `TestBed` from `@angular/core/testing`; import `AppComponent` from `./app.component`; import `RouterTestingModule` or use `provideRouter([])` in `TestBed.configureTestingModule` providers; write one `it('should create the app', ...)` test that creates `AppComponent` via `TestBed.createComponent` and asserts `fixture.componentInstance` is truthy.
