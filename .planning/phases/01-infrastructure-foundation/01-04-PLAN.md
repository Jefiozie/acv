<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Lambda execution role → DynamoDB | Lambda assumes IAM role; must be least-privilege (read/write only, not admin) |
| API Gateway → Lambda | Public internet hits API Gateway; Lambda is the trust boundary for input validation (Phase 2) |
| Lambda bundling (esbuild) → npm packages | Bundled packages execute in Lambda; supply chain risk for @aws-sdk/* and zod |
| EventBridge Rule → Lambda | EventBridge invokes Lambda on schedule; Lambda resource policy must allow only the specific rule ARN |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-09 | Elevation of Privilege | Lambda IAM execution role | mitigate | Use `table.grantReadWriteData(fn)` — CDK generates least-privilege IAM policy scoped to the specific table ARN and DynamoDB actions (GetItem, PutItem, UpdateItem, DeleteItem, Query, Scan); never use managed policies like AmazonDynamoDBFullAccess |
| T-01-10 | Tampering | Lambda environment variables | mitigate | TABLE_NAME, SES_FROM_ADDRESS, SES_CONFIG_SET passed via CDK construct props — never hardcoded strings; values come from StatefulStack outputs at deploy time |
| T-01-11 | Denial of Service | EventBridge Rule (unbounded invocations) | accept | Rate(10 min) is bounded; Lambda timeout at 5 min prevents runaway execution; CloudWatch Errors alarm planned for Phase 4 |
| T-01-12 | Information Disclosure | API Gateway URL in CDK output | accept | URL is non-guessable (random ID) but is not a secret; CORS restricts browser access to known origins; Phase 4 adds WAF if needed |
| T-01-13 | Tampering | AWS SDK bundling (no externalModules) | mitigate | Bundling SDK at synth time locks to a specific known-good version; avoids version drift from Lambda runtime updates; verified by `grep externalModules` check in Task 1 |
| T-01-SC | Tampering | npm installs | accept | No new package installs in this plan; packages already verified in 01-01 Task 1 |
</threat_model>
