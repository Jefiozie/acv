## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-02-01 | Spoofing / XSS | `subscribe.component.html` error messages | mitigate | All user-controlled values rendered via Angular's `{{ }}` interpolation, which HTML-escapes by default. No `[innerHTML]` used anywhere on user data or API error responses |
| T-03-02-02 | Information Disclosure | API error handling in `onSubmit()` | mitigate | The `error` state shows a fixed Dutch string; the actual `HttpErrorResponse` message, status code, or server body is never passed to the template or logged to the console in production |
| T-03-02-03 | Tampering | Client-side form validation bypass | accept | Signal Forms validation is UX-only; Phase 2 backend re-validates all inputs server-side with zod. Client-side validation does not replace server-side validation |
| T-03-02-04 | Elevation of Privilege | CSRF on POST /api/subscribe | accept | No authentication tokens in this flow; the subscribe endpoint is intentionally public. CORS policy (configured in Phase 2 API Gateway) is the server-side control |
