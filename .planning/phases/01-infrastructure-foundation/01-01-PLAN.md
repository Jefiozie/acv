    STEP B — Update .gitignore (append if existing, create if missing):

      `cd infrastructure && npx cdk bootstrap aws://$CDK_DEFAULT_ACCOUNT/$AWS_REGION`

      `cd infrastructure && npx cdk deploy AcvStateful`
      `cd infrastructure && npx cdk deploy AcvBackend`
    Note: Run from inside `infrastructure/` directory. Never use root credentials for CDK;
    use a scoped IAM user/role.

    STEP D — Create infrastructure/package.json:
    Name: `@acv/infrastructure`. Set `"private": true`. Add devDependencies:
    aws-cdk-lib@2.257.0, constructs@10.6.0, esbuild (latest), ts-node (latest),
    typescript (^5.8.0), @types/aws-lambda (latest), @types/node (^22.0.0).

    Do NOT add these as `dependencies` — CDK packages are build-time only.

    STEP E — Create infrastructure/tsconfig.json:
    Must extend `../tsconfig.json`. Add compilerOptions: module CommonJS,
    moduleResolution node, outDir `cdk.out/ts`, rootDir `.`, sourceMap true,
    esModuleInterop true, experimentalDecorators true, resolveJsonModule true.
    Include: `["bin/**/*", "lib/**/*"]`.
    Exclude: `["node_modules", "cdk.out"]`.

    STEP F — Create infrastructure/cdk.json:
    Set `"app"` to `"npx ts-node --prefer-ts-exts bin/app.ts"`.
    Set `"watch"` to empty object for now.
    Set `"context"` to include `"@aws-cdk/aws-apigateway:usagePlanKeyOrderInsensitiveId": true`
    and `"@aws-cdk/core:stackRelativeExports": true` (standard CDK feature flags).
    Do NOT include AWS account IDs or region in cdk.json — use CDK_DEFAULT_ACCOUNT and
    CDK_DEFAULT_REGION environment variables instead.

    STEP G — Create infrastructure/bin/app.ts (complete wiring — not modified in later plans):
    Import aws-cdk-lib as `cdk`. Import StatefulStack from `../lib/stateful-stack`.
    Import BackendStack from `../lib/backend-stack`.
    Create `const app = new cdk.App()`.
    Create `const env` object using `process.env.CDK_DEFAULT_ACCOUNT` and
    `process.env.CDK_DEFAULT_REGION ?? 'eu-west-1'`. NEVER hardcode account ID or region.
    Instantiate `new StatefulStack(app, 'AcvStateful', { env, terminationProtection: true })`.
    Store in const `stateful`. Pass `stateful.table`, `stateful.configSetName`, and
    `sesFromAddress: 'noreply@acv-aanhanger.nl'` to `new BackendStack(app, 'AcvBackend', { env, ... })`.

    STEP H — Create infrastructure/lib/stateful-stack.ts (typed stub — filled in plan 01-02):
    Declare StatefulStackProps extending cdk.StackProps (empty for now).
    Export class StatefulStack extending cdk.Stack with:
    - `public readonly table!: dynamodb.TableV2` (definite assignment assertion — set by 01-02)
    - `public readonly configSetName!: string`
    Constructor calls `super(scope, id, props)` and adds a comment `// Full implementation: plan 01-02`.
    Import dynamodb from `aws-cdk-lib/aws-dynamodb`.

    STEP I — Create infrastructure/lib/backend-stack.ts (typed stub — filled in plan 01-04):

    table (dynamodb.TableV2), sesConfigSetName (string), sesFromAddress (string).
    Export class BackendStack extending cdk.Stack. Constructor calls `super(scope, id, props)`.
    Add comment `// Full implementation: plan 01-04`.

    `cd infrastructure && npm install`
